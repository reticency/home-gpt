## 1. 产品概述

### 1.1 产品定位

**HomeGPT** —— 装修后悔预防与家庭生活设计助手。

基于 AI Agent 的家庭空间设计助手，帮助用户从"装修小白"成长为"家庭生活设计师"，减少装修后悔，提升居住幸福感。

### 1.2 使命

帮助用户设计未来十年的生活，而非仅仅装修房子。

### 1.3 一句话描述

上传户型图，AI 告诉你这个家应该怎么设计。

---

## 2. 目标用户

| 用户群体     | 核心需求                     | 使用场景                 |
| ------------ | ---------------------------- | ------------------------ |
| 首次装修家庭 | 缺乏装修决策经验，避免踩坑   | 购买新房准备装修         |
| 新婚夫妻     | 共同规划未来家居空间         | 婚房装修                 |
| 有孩子的家庭 | 儿童成长空间规划、安全性考虑 | 儿童房设计、家庭空间改造 |
| 老房翻新人群 | 空间利用率提升、功能优化     | 旧房改造升级             |

---

## 3. 用户痛点

- ❌ 花费大量时间浏览装修案例、咨询设计师
- ❌ 方案反复修改，沟通成本高
- ❌ 被各种风格和营销信息影响，决策困难
- ❌ 收纳不足、空间利用率低、预算失控
- ❌ 效果图好看但实际入住后不好用

---

| 维度     | 价值描述                                               |
| -------- | ------------------------------------------------------ |
| 商业价值 | 服务庞大的家装市场，提供低门槛、个性化的装修决策服务   |
| 效率提升 | 快速完成装修方案分析，减少试错成本，提高决策效率       |
| 社会价值 | 降低装修焦虑，让用户从"装修小白"成长为"家庭生活设计师" |

## 5. 核心功能

### 5.1 功能优先级

| 优先级 | 功能模块   | 功能点                     | 说明                                           |
| ------ | ---------- | -------------------------- | ---------------------------------------------- |
| **P0** | 户型图上传 | 图片/PDF/截图上传          | 支持主流格式，Base64 本地存储                  |
| **P0** | 需求填写   | 家庭成员、预算、风格、需求 | 仅保留四个核心字段                             |
| **P0** | Agent 分析 | 自动召集专家分析           | 户型分析师、家庭规划师、收纳设计师、装修预算师 |
| **P0** | 结果展示   | 一页式报告输出             | 评分、优缺点、建议、预算分配、效果图           |

### 5.2 MVP 范围

✅ 包含：
- 户型图上传（图片/PDF/截图）
- 需求填写（家庭成员、预算、风格、需求）
- Agent 自动分析（专家召集展示）
- 一页式结果输出（评分、分析、建议、预算、效果图）

❌ 不包含：
- 登录注册
- 社区功能
- 收藏功能
- VR/全屋漫游
- 多房间设计
- AI 对话
- 家具商城

## 8. 功能详细设计

### 8.1 首页（Landing Page）

**页面位置**：`/`

**页面类型**：Server Component

**设计风格**：极简、留白、温暖的中性色调（参考苹果官网风格）

**交互元素**：
| 元素           | 说明                                      | 交互                 |
| -------------- | ----------------------------------------- | -------------------- |
| Logo + 品牌名  | HomeGPT                                   | 点击返回首页         |
| 主标题         | "设计你未来想要的生活"                    | 静态展示             |
| 副标题         | "上传户型图，AI 告诉你这个家应该怎么设计" | 静态展示             |
| CTA 按钮       | "开始设计"                                | 点击跳转至 `/upload` |
| 功能介绍卡片   | 智能户型分析、家庭规划师、AI 效果图       | 静态展示             |
| 为什么选择我们 | 4 个核心优势 + 数据展示                   | 静态展示             |
| 底部 CTA       | "准备好设计你的理想家了吗？"              | 点击跳转至 `/upload` |
| 页脚           | 版权信息                                  | 静态展示             |

### 8.2 上传页面（Step 1）

**页面位置**：`/upload`

**页面类型**：Client Component

**交互元素**：
| 元素       | 说明                           | 交互                       |
| ---------- | ------------------------------ | -------------------------- |
| 步骤指示器 | 1/4 上传户型图                 | 静态展示                   |
| 标题       | "上传你的户型图"               | 静态展示                   |
| 上传区域   | 支持拖放和点击上传             | 拖放/选择文件触发上传      |
| 格式提示   | 支持图片 (JPG/PNG)、PDF、截图  | 静态展示                   |
| 文件预览   | 图片显示预览，PDF 显示文件图标 | 自动生成预览               |
| 下一步按钮 | "下一步"                       | 点击跳转至 `/requirements` |
| 重新上传   | 删除按钮                       | 清除当前文件               |

**业务规则**：
- 文件类型限制：JPG、PNG、PDF
- 文件大小限制：单文件不超过 10MB
- 图片自动压缩后存储

### 8.3 需求填写页面（Step 2）

**页面位置**：`/requirements`

**页面类型**：Client Component

**交互元素**：
| 元素       | 类型         | 选项/说明                                      |
| ---------- | ------------ | ---------------------------------------------- |
| 步骤指示器 | 2/4 填写需求 | 静态展示                                       |
| 家庭成员   | 多选标签     | 单身、夫妻、孩子(数量)、老人、宠物             |
| 预算       | 数字输入     | 单位：万元                                     |
| 风格       | 单选卡片     | 原木风、现代简约、北欧风、轻奢风、新中式、日式 |
| 需求       | 多选标签     | 大量收纳、亲子互动、工作学习、休闲娱乐、无障碍 |
| 下一步按钮 | "下一步"     | 点击跳转至 `/analyzing`                        |

**业务规则**：
- 所有字段为必填项
- 预算必须为有效数字
- 表单未填写完整时，下一步按钮禁用

### 8.4 分析中页面（Step 3）

**页面位置**：`/analyzing`

**页面类型**：Client Component

**交互元素**：
| 元素       | 说明                                           | 交互         |
| ---------- | ---------------------------------------------- | ------------ |
| 步骤指示器 | 3/4 分析中                                     | 静态展示     |
| 标题       | "HomeGPT 正在召集专家……"                       | 静态展示     |
| 专家列表   | 户型分析师、家庭规划师、收纳设计师、装修预算师 | 逐一点亮动画 |
| 进度条     | 加载动画                                       | 自动进度展示 |

**业务规则**：
- 自动依次显示专家就位状态（带勾选动画）
- 全部完成后自动跳转至结果页面（约 5-8 秒）
- 触发 AI 分析 API 和效果图生成 API

### 8.5 结果页面（Step 4）

**页面位置**：`/result`

**页面类型**：Client Component

**交互元素**：
| 元素         | 说明                    | 交互                 |
| ------------ | ----------------------- | -------------------- |
| 步骤指示器   | 4/4 结果                | 静态展示             |
| Home 评分    | 0-100 分                | 静态展示             |
| 户型分析     | 优点列表、缺点列表      | 静态展示             |
| AI 决策建议  | 建议列表                | 静态展示             |
| 预算建议     | 柜体/硬装/软装/预留比例 | 静态展示             |
| AI 效果图    | 单张效果图              | 静态展示             |
| 重新设计按钮 | "重新设计"              | 点击跳转至 `/upload` |
| 保存方案按钮 | "保存方案"              | 保存到 localStorage  |

**业务规则**：
- 结果数据存储到 localStorage
- 效果图使用 AI 生成（DALL-E）

## 9. 数据结构

### 9.1 localStorage 存储键名

| 键名                       | 用途                 |
| -------------------------- | -------------------- |
| `homegpt_current_project`  | 当前进行中的项目数据 |
| `homegpt_projects`         | 历史项目列表         |
| `homegpt_user_preferences` | 用户偏好设置         |

### 9.2 类型定义

```typescript
export interface FloorPlan {
  base64: string;
  filename: string;
  type: 'image' | 'pdf';
  size: number;
}

export interface FamilyMember {
  type: 'single' | 'couple' | 'child' | 'elder' | 'pet';
  count?: number;
}

export interface UserRequirement {
  type: 'storage' | 'parenting' | 'work' | 'entertainment' | 'accessibility';
}

export interface ProjectData {
  id: string;
  floorPlan: FloorPlan;
  familyMembers: FamilyMember[];
  budget: number;
  style: 'wood' | 'modern' | 'nordic' | 'luxury' | 'chinese' | 'japanese';
  requirements: UserRequirement[];
  status: 'pending' | 'analyzing' | 'complete' | 'failed';
  createdAt: number;
}

export interface AnalysisResult {
  projectId: string;
  score: number;
  advantages: string[];
  disadvantages: string[];
  suggestions: string[];
  budgetAllocation: {
    cabinet: number;
    hardware: number;
    soft: number;
    reserve: number;
  };
  renderingUrl: string;
  analyzedAt: number;
}
```

## 10. 技术方案

### 10.1 技术栈

| 层级     | 技术                    | 说明                                   |
| -------- | ----------------------- | -------------------------------------- |
| 框架     | Next.js 14 (App Router) | 全栈框架，支持 Server/Client Component |
| 语言     | TypeScript              | 类型安全                               |
| 样式     | Tailwind CSS 3          | 快速构建，设计系统                     |
| 图标     | lucide-react            | 简洁图标                               |
| 状态管理 | React Context           | 轻量级状态管理                         |
| 存储     | localStorage            | 客户端本地存储                         |
| AI 服务  | OpenAI API              | Agent 分析和效果图生成                 |

### 10.2 API 设计

#### 10.2.1 触发分析

```
POST /api/analyze

Body:
{
  "projectId": "string",
  "floorPlanBase64": "string",
  "familyMembers": [{"type": "couple"}, {"type": "child", "count": 1}],
  "budget": 18,
  "style": "wood",
  "requirements": [{"type": "storage"}]
}

Response:
{
  "success": true,
  "result": {
    "score": 87,
    "advantages": ["南北通透", "动静分离"],
    "disadvantages": ["主卧收纳不足", "入户浪费空间"],
    "suggestions": ["玄关做到顶柜", "餐边柜增加收纳", ...],
    "budgetAllocation": {"cabinet": 35, "hardware": 40, "soft": 15, "reserve": 10}
  }
}
```

#### 10.2.2 生成效果图

```
POST /api/generate

Body:
{
  "projectId": "string",
  "style": "wood",
  "description": "原木风客厅，南北通透，餐边柜收纳"
}

Response:
{
  "success": true,
  "renderingUrl": "https://oaidalleapiprodscus.blob.core.windows.net/private/..."
}
```

## 12. 验收标准

### 12.1 功能验收

| 编号 | 验收项     | 验收标准                                                     |
| ---- | ---------- | ------------------------------------------------------------ |
| F1   | 户型图上传 | 用户可以上传图片（JPG/PNG）和 PDF 文件                       |
| F2   | 文件预览   | 图片显示预览，PDF 显示文件图标卡片                           |
| F3   | 需求填写   | 用户可以填写家庭成员、预算、风格、需求四个字段               |
| F4   | 表单验证   | 必填项未填写时下一步按钮禁用                                 |

| F6   | 结果展示   | 展示一页式分析报告，包含评分、优缺点、建议、预算分配、效果图 |
| F7   | 重新设计   | 用户可以点击重新设计返回上传页面                             |
| F8   | 数据存储   | 项目数据和分析结果存储到 localStorage                        |
