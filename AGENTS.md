# HomeGPT AGENTS.md

---

## 1. 项目背景和核心功能描述

### 1.1 项目背景

**HomeGPT** 是一款基于 AI Agent 的家庭空间设计助手，旨在帮助普通家庭解决"装修效果图好看，但实际入住后不好用"的痛点。

目标用户：首次装修家庭、新婚夫妻、有孩子的家庭、老房翻新人群。

核心问题：用户缺少专业的装修决策能力，而非设计灵感。

使命：帮助用户设计未来想要的生活，而非仅仅装修房子。

### 1.2 核心功能

MVP 版本仅包含四个步骤：

| 步骤   | 功能       | 说明                                                         |
| ------ | ---------- | ------------------------------------------------------------ |
| Step 1 | 上传户型图 | 支持图片/PDF，Base64 本地存储                                |
| Step 2 | 填写需求   | 家庭成员、预算、风格、需求（四个字段）                       |
| Step 3 | Agent 分析 | 自动召集专家：户型分析师、家庭规划师、收纳设计师、装修预算师 |
| Step 4 | 结果展示   | 一页式报告：评分、优缺点、建议、预算分配、AI 效果图          |

### 1.3 技术栈

| 层级     | 技术                    |
| -------- | ----------------------- |
| 框架     | Next.js 14 (App Router) |
| 语言     | TypeScript              |
| 样式     | Tailwind CSS 3          |
| 图标     | lucide-react            |
| 状态管理 | React Context           |
| 存储     | localStorage            |
| AI 服务  | OpenAI API              |

## 2. 代码风格规范

### 2.1 TypeScript 规范

- **严格模式**：始终使用 `strict: true`，启用所有严格类型检查选项
- **类型定义**：为所有函数参数、返回值和组件 props 定义类型
- **避免 any**：禁止使用 `any` 类型，使用 `unknown` + 类型守卫替代
- **接口优先**：使用 `interface` 定义数据结构，使用 `type` 定义联合类型
- **泛型使用**：合理使用泛型提高代码复用性
- **枚举使用**：使用 `const` 对象替代 TypeScript 枚举

```typescript
// 推荐
export const StyleType = {
  WOOD: 'wood',
  MODERN: 'modern',
  NORDIC: 'nordic',
} as const;

export type StyleType = typeof StyleType[keyof typeof StyleType];

// 不推荐
enum StyleType {
  Wood = 'wood',
  Modern = 'modern',
}
```

### 2.2 React 规范

- **组件命名**：使用 PascalCase，文件同名导出
- **Props 定义**：使用 `interface Props` 定义组件属性
- **Hooks 规则**：严格遵守 Hooks 规则，只在组件顶层调用
- **自定义 Hooks**：以 `use` 开头命名
- **Client Component**：使用 `"use client"` 指令标识客户端组件
- **Server Component**：默认即为服务端组件，无需额外标记

```tsx
// Client Component 示例
'use client';

import { useState } from 'react';

interface CounterProps {
  initialValue?: number;
}

export function Counter({ initialValue = 0 }: CounterProps) {
  const [count, setCount] = useState(initialValue);
  return (
    <button onClick={() => setCount(c => c + 1)}>
      Count: {count}
    </button>
  );
}
```

### 2.3 文件组织

- **路由页面**：`app/` 目录下按路由组织，使用 `page.tsx` 作为页面入口
- **组件**：`components/` 目录存放可复用组件
- **上下文**：`context/` 目录存放 React Context
- **Hooks**：`hooks/` 目录存放自定义 Hooks
- **类型**：`types/` 目录存放 TypeScript 类型定义
- **工具函数**：`utils/` 目录存放通用工具函数
- **API 路由**：`app/api/` 目录存放服务端 API

### 2.4 代码格式

- **缩进**：使用 2 空格缩进
- **分号**：使用分号结尾
- **引号**：使用单引号，JSX 中使用双引号
- **换行**：每行不超过 100 字符
- **空行**：函数之间、逻辑块之间保留适当空行
- **注释**：仅在必要时添加注释，代码应自解释

## 3. UI/UX 设计原则

### 3.1 整体风格

- **极简主义**：参考苹果官网风格，大量留白，干净利落
- **温暖色调**：使用中性色为主（`#F5F5F7` 背景、`#1D1D1F` 文字）
- **圆润设计**：按钮和卡片使用圆角（8-12px）
- **阴影克制**：使用微妙的阴影增加层次感，不使用强烈阴影
- **紫色渐变禁令**：严禁使用科技感强烈的紫色渐变

### 3.2 颜色规范

| 颜色     | Hex       | 用途             |
| -------- | --------- | ---------------- |
| 背景色   | `#F5F5F7` | 页面背景         |
| 主文字   | `#1D1D1F` | 标题、正文       |
| 次要文字 | `#6E6E73` | 辅助说明         |
| 主按钮   | `#0071E3` | CTA 按钮         |
| 成功色   | `#34C759` | 勾选、成功状态   |
| 错误色   | `#FF3B30` | 叉号、错误状态   |
| 边框色   | `#E5E5EA` | 分隔线、卡片边框 |

### 3.3 字体规范

- **主标题**：SF Pro Display, 48px, 600
- **副标题**：SF Pro Display, 24px, 400
- **正文**：SF Pro Text, 17px, 400
- **辅助文字**：SF Pro Text, 14px, 400

### 3.4 交互原则

- **即时反馈**：按钮点击有视觉反馈，表单输入有即时验证
- **加载状态**：异步操作显示加载动画，避免用户困惑
- **平滑过渡**：页面切换和状态变化使用平滑动画
- **清晰流程**：四步流程使用步骤指示器，让用户明确当前位置
- **一键操作**：核心操作（开始设计、重新设计）突出显示

### 3.5 响应式设计

- **移动端优先**：确保在手机端有良好体验
- **断点适配**：使用 Tailwind 默认断点（sm: 640px, md: 768px, lg: 1024px）
- **触控友好**：按钮最小尺寸 44px，适合手指点击
- **布局自适应**：内容在不同屏幕尺寸下合理排列

## 4. 错误处理规则

### 4.1 localStorage 安全访问

**必须**在所有 localStorage 操作前检查 `typeof window !== 'undefined'`，避免服务端渲染时出错。

```typescript
// 推荐
export const getStorageData = (): HomeGPTData => {
  if (typeof window === 'undefined') {
    return defaultData;
  }
  const data = localStorage.getItem(STORAGE_KEY);
  return data ? JSON.parse(data) : defaultData;
};

// 不推荐
export const getStorageData = (): HomeGPTData => {
  const data = localStorage.getItem(STORAGE_KEY); // SSR 环境会报错
  return data ? JSON.parse(data) : defaultData;
};
```

### 4.2 文件上传错误处理

- **文件类型校验**：仅允许 JPG/PNG/PDF 格式
- **文件大小限制**：单文件不超过 10MB
- **压缩处理**：图片自动压缩后存储，避免 localStorage 超限
- **错误提示**：友好的错误提示信息，指导用户正确操作

### 4.3 API 错误处理

- **网络错误**：显示网络异常提示，提供重试按钮
- **超时处理**：设置合理超时时间（10-15 秒），超时后提示用户
- **API Key 缺失**：服务端返回明确错误，前端提示配置问题
- **降级策略**：AI 服务不可用时，使用预设的模拟数据进行演示

### 4.4 表单验证

- **必填项检查**：所有必填字段必须填写才能继续
- **格式验证**：预算输入必须为有效数字
- **实时反馈**：输入时即时验证，显示错误提示
- **禁用状态**：表单未填写完整时，下一步按钮禁用

### 4.5 错误日志

- **开发环境**：详细的控制台日志输出
- **生产环境**：静默失败，友好提示用户
- **错误边界**：使用 React Error Boundary 捕获组件错误

## 5. 不应该做的事情（行为约束）

### 5.1 功能范围约束

- ❌ **不要实现登录注册**：MVP 版本不需要用户系统
- ❌ **不要实现社区功能**：评论、分享等社交功能不在 MVP 范围内
- ❌ **不要实现收藏功能**：无需保存历史方案（localStorage 自动处理）
- ❌ **不要实现 VR/全屋漫游**：技术复杂度高，不符合 MVP 定位
- ❌ **不要实现多房间设计**：单张效果图足够演示
- ❌ **不要实现 AI 对话**：避免多轮对话，保持四步流程
- ❌ **不要实现家具商城**：电商功能超出当前范围

### 5.2 技术约束

- ❌ **不要使用数据库**：使用 localStorage 存储，无需后端数据库
- ❌ **不要使用复杂状态管理**：使用 React Context 即可，无需 Redux/Zustand
- ❌ **不要使用服务端渲染存储**：localStorage 仅在客户端可用
- ❌ **不要过度设计**：保持简单，满足需求即可
- ❌ **不要引入不必要的依赖**：每个依赖都应有明确用途

### 5.3 设计约束

- ❌ **不要使用紫色渐变**：保持极简风格，避免科技感强烈的配色
- ❌ **不要使用复杂动画**：动画应简洁克制，不干扰用户操作
- ❌ **不要使用过度装饰**：避免多余的视觉元素，保持干净
- ❌ **不要使用花哨字体**：使用系统字体 SF Pro，保持专业感
- ❌ **不要使用强烈对比色**：保持温和的色彩搭配

### 5.4 代码约束

- ❌ **不要写未使用的代码**：删除死代码，保持代码整洁
- ❌ **不要写注释**：代码应自解释，无需额外注释
- ❌ **不要使用 console.log**：生产代码中移除所有 console 语句
- ❌ **不要忽略 TypeScript 错误**：所有类型错误必须修复
- ❌ **不要重复代码**：提取公共逻辑为组件或工具函数
- ❌ **不要破坏现有功能**：修改时确保不影响已实现的功能
